## 文档更新规范

1. 版本号规则

   - 主版本号：重大架构变更
   - 次版本号：功能模块变更
   - 修订号：细节优化

2. 命名规范

   - 文件名：功能名\_v 版本号.md
   - 版本号：v1, v2, v3...

3. 更新流程
   - 创建新版本文档
   - 更新 README.md
   - 提交 git 记录
   - 通知团队成员

## 注意事项

1. 保持文档的连续性
2. 及时更新 README.md
3. 重要变更需要团队确认
4. 定期检查文档时效性
